
# Output Directory README

This folder contains the output files generated during the data cleaning and exploratory data analysis phases of the project. The files include filtered datasets, statistical summaries, correlation tables, and visual reports.

---

## 📂 Directory Contents

### 1. Data Cleaning Outputs (`01_data_cleaning`)

* **`dropped_patients_fitbit_log.txt`**

  * **Source Script:** `preprocess_fitbit.py`
  * **Description:** Contains a table of patient IDs excluded from the Fitbit dataset. It logs the maximum number of consecutive days with missing data (per feature) for each dropped patient.
  * **Exclusion Criteria:** Patients were dropped if their maximum number of consecutive missing days for any feature was $\ge 2$.
* **`sum_placental_histo_features.txt`**

  * **Source Script:** `preprocess_placental.py`
  * **Description:** Contains the sum of values for each column corresponding to a placental histopathology feature (calculated after features were encoded to numerical values: 0, 1, 2, 3, etc.). Columns were automatically dropped from the final dataset if the feature was not present in any patient.

### 2. Exploratory Analysis Outputs (`02_exploratory_analysis`)

#### Fitbit Statistics & Visualizations

* **`fitbit_data_analysis.txt`**

  * **Source Script:** `analyze_fitbit.py`
  * **Description:** Comprehensive statistical summary of the Fitbit dataset, strictly filtered to include only datapoints during pregnancy. It includes:

    * Total number of missing values across the dataset.
    * A table of missing days per patient (a day is counted as missing only if *all* metrics are absent).
    * A table tracking the maximum consecutive missing days per feature per patient.
    * Total number of unique dates recorded across the cohort.
    * Summary statistics (Median and IQR) broken down by metric.
* **`pregnancy_plots_report.pdf`**

  * **Source Script:** `histogram.py`
  * **Description:** A PDF report containing two comparative histograms:

    1. **All Valid Datapoints:** Plots the number of patients per day using all valid Fitbit updates, featuring vertical dotted lines to mark pregnancy trimesters and typical delivery date.
    2. **Pregnancy-Only Datapoints:** A similar histogram, but filtered to display only data captured during pregnancy.

#### Correlation Analysis Tables

> **Note on Correlation Methodology:** All correlation files below are generated by `correlation.py` and evaluate the relationship between two distinct sets of variables: **Placental Histopathology Variables** and **Delivery Variables** (e.g., baby height/weight, gestational age, Apgar scores at 1/5 minutes, NICU days). Correlations are only calculated *between* these two sets, not within them. Each entry records $\rho$ (rho), $p$-value, sample size ($N$), and False Discovery Rate (FDR).

* **`full_correlation_table.txt`**

  * **Description:** The complete, unfiltered master table containing the correlation measurements for every pair between the two variable sets.
* **`filtered_correlation_table.txt`**

  * **Description:** A filtered subset of the master table, retaining only the variable pairs that successfully passed the FDR significance test.
* **`positively_associated_delivery_vars.txt`**

  * **Description:** Contains all significant delivery variables that showed a *positive* association with placental features ($\rho > 0$ and passed the FDR test).
* **`negatively_associated_delivery_vars.txt`**

  * **Description:** Contains all significant delivery variables that showed a *negative* association with placental features ($\rho < 0$ and passed the FDR test).
