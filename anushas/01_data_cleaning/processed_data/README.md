# processed_data

This folder stores the cleaned and harmonized datasets used for downstream analysis in the DP3 project. These files are generated by the preprocessing scripts in the neighboring data-cleaning pipeline and are intended to be read by later exploratory and modeling workflows.

## Files in this folder

- `processed_clinical_data.csv`

  - Cleaned clinical data derived from the raw clinical table.
  - Generated by `01_data_cleaning/preprocess_clinical.py`.
- `processed_fitbit_data.csv`

  - Cleaned Fitbit-derived longitudinal data with standardized column names and missing-data filtering applied.
  - Generated by `01_data_cleaning/preprocess_fitbit.py`.
- `processed_placental_data.csv`

  - Cleaned placental histopathology data with standardized labels and encoded categorical findings.
  - Generated by `01_data_cleaning/preprocess_placental.py`.
- `master_fitbit_clinical_correlation_data.csv`

  - A merged analysis-ready table that combines trimester-level Fitbit summaries with clinical and placental variables.
  - Generated by `01_data_cleaning/preprocess_correlation.py`.

## What each preprocessing step does

### 1. Clinical preprocessing

Script: `01_data_cleaning/preprocess_clinical.py`

This workflow:

- loads the raw clinical spreadsheet,
- standardizes column names and text values,
- filters for delivered patients,
- converts missing-value placeholders to `NaN`,
- adds missingness indicators,
- encodes smoking status and demographic features,
- and writes the final clinical table to `processed_clinical_data.csv`.

### 2. Fitbit preprocessing

Script: `01_data_cleaning/preprocess_fitbit.py`

This workflow:

- reads the Fitbit source files,
- harmonizes column names across sheets,
- merges the data on `record_id` and `date`,
- sorts the columns into a consistent layout,
- and nullifies metrics for patients with long consecutive gaps in a feature series.

### 3. Placental preprocessing

Script: `01_data_cleaning/preprocess_placental.py`

This workflow:

- combines the two placental source sheets,
- removes patients without usable slides,
- drops non-informative columns,
- encodes categorical pathology findings into numeric values,
- and writes the cleaned placental dataset.

### 4. Correlation-ready integration

Script: `01_data_cleaning/preprocess_correlation.py`

This workflow:

- loads the three cleaned datasets,
- filters Fitbit rows to pregnancy-related events,
- bins rows into trimester-based groups,
- averages relevant metrics by patient and trimester,
- merges the resulting Fitbit summaries with clinical and placental variables,
- and writes the final master correlation table.

## How to regenerate the outputs

From the `anushas` project directory, run:

```bash
python 01_data_cleaning/preprocess_clinical.py
python 01_data_cleaning/preprocess_fitbit.py
python 01_data_cleaning/preprocess_placental.py
python 01_data_cleaning/preprocess_correlation.py
```

The correlation script depends on the three cleaned intermediate files being present, so it should be run after the first three scripts complete successfully.
